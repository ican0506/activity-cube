# 活动魔方测试说明

## 测试前准备

1. 在 Navicat 执行 `sql/activity_cube_final.sql` 初始化全新数据库，或对已有库执行 `docs/sql` 下对应补丁。
2. 启动后端：`cd backend && mvn spring-boot:run`。
3. 启动前端：`cd frontend && npm run dev`。
4. 测试账号：
   - 学生：`2321241389 / 123456`
   - 负责人：`T2024001 / 123456`
   - 管理员：`admin / 123456`

## 1. 学生注册登录

1. 打开 `/register`。
2. 填写姓名、10 位学号、密码、校区、学院、专业、班级、手机号。
3. 提交后确认自动登录或可用学号登录。
4. 使用 `role=admin` 或 `role=organizer` 抓包提交时，后端仍应创建 `student` 角色。

## 2. 负责人创建活动

1. 使用负责人账号登录。
2. 进入 `/admin/activities`，创建线下或线上活动。
3. 保存草稿后确认学生端活动大厅不可见。
4. 提交审核后确认状态为“待审核”。

## 3. 管理员创建负责人账号

1. 使用管理员账号登录。
2. 进入用户管理页面，创建负责人账号，填写姓名、工号、初始密码、校区、部门。
3. 退出后使用该工号和初始密码登录。
4. 使用学生账号访问创建负责人接口，应返回无权限。

## 4. 学生报名

1. 管理员审核通过活动。
2. 学生进入 `/activities`，打开活动详情。
3. 在报名时间内点击“立即报名”并提交。
4. 重复报名应提示“你已报名，请勿重复提交”。
5. 人数已满时应提示“该活动人数已满”。

## 5. 线上活动签到

1. 创建并审核一个 `online` 活动。
2. 报名后在签到时间范围内进入活动详情。
3. 点击“去签到”，进入签到确认页。
4. 点击“确认签到”，应提示签到成功。
5. 重复签到应提示“你已完成签到，请勿重复签到”。

## 6. 线下活动扫码签到

1. 创建并审核一个 `offline` 活动。
2. 负责人进入 `/admin/activities/:id/qrcodes` 生成签到二维码。
3. 学生直接从活动详情进入签到入口时，应看到“线下活动需到现场扫描签到二维码”。
4. 使用 `/scan` 扫码、粘贴签到二维码链接，或输入现场签到码。
5. 进入签到确认页后点击“确认签到”，应完成签到。
6. 缺少或错误签到码时，应提示“线下活动需扫描现场签到二维码”或“签到二维码无效”。

## 7. 人工补签

1. 学生先完成报名但不签到。
2. 负责人进入该活动签到名单页面，点击“人工补签”。
3. 选择已报名未签到学生，填写原因，提交。
4. 签到名单显示“人工补签”、操作人和备注。
5. 未报名学生不能补签；已签到学生不能重复补签。
6. 负责人不能给别人创建的活动补签；管理员可以给任意活动补签。

## 8. 活动反馈

1. 活动结束后，学生进入活动详情，点击“活动反馈”。
2. 提交评分、体验、建议和附件。
3. 同一用户再次提交应提示“你已提交过反馈”。
4. 管理端反馈统计页显示平均分、评分分布、文字反馈和附件。

## 9. 消息中心

1. 学生报名成功后进入 `/messages`。
2. 应看到“报名成功”通知，未读角标增加。
3. 打开消息详情后自动标记已读。
4. 点击“一键全部已读”，未读数应归零。
5. 负责人发布活动通知后，目标学生应收到消息。
6. 管理员发布系统通知后，学生应收到系统通知。

## 10. 导出名单

1. 负责人或管理员进入报名名单页面，点击导出报名名单。
2. 进入签到名单页面，点击导出签到名单。
3. 进入统计页面，点击导出未签到名单。
4. 文件名应包含活动名称和名单类型。
5. 学生直接调用导出接口应返回无权限。

## 11. 权限测试

1. 未登录访问 `/activities/:id/register`、`/activities/:id/checkin` 应跳转登录页，并保留 redirect。
2. 学生访问 `/admin/**` 应返回无权限或被前端拦截。
3. 负责人只能编辑、删除、导出、补签、发布通知自己创建的活动。
4. 管理员可以管理所有活动、用户、审核、操作日志和系统通知。
5. 401 返回后前端应清除 token 并跳转登录页。
