# 活动魔方前端 UI 重设计 AI 提示词

请你作为一名前端 UI/UX 设计与 Vue 3 开发专家，帮助我重设计并优化“活动魔方”项目前端界面。

## 一、项目背景

项目名称：活动魔方

项目定位：面向高校校园活动组织的一站式轻工具平台。

使用场景：

- 校园微服务平台
- 微信公众号 H5 应用
- 电脑端后台管理
- 手机端学生活动入口

目标学校场景：河南农业大学。

支持校区：

1. 龙子湖校区
2. 文化路校区
3. 许昌校区

项目已有功能：

1. 活动创建
2. 活动报名
3. 活动签到
4. 二维码报名
5. 二维码签到
6. 报名名单查看
7. 签到名单查看
8. Excel 导出
9. 随机抽签
10. 随机分组
11. 活动反馈
12. 满意度统计
13. 图片上传
14. 视频上传

## 二、技术栈

前端技术栈：

- Vue 3
- Vite
- Element Plus
- Axios
- Pinia
- ECharts
- xlsx
- qrcode

请不要引入复杂 UI 框架，不要重构项目架构，不要破坏已有接口和业务逻辑。

## 三、当前问题

当前项目功能基本完成，但存在以下问题：

1. 页面视觉不够统一；
2. 部分页面像普通后台管理模板；
3. 缺少校园活动平台的产品感；
4. 活动大厅、活动详情、抽签分组、反馈统计等页面展示效果不够强；
5. 手机端虽然能显示，但还不够适合微信公众号 H5 使用。

## 四、设计目标

请按以下目标重设计前端 UI：

1. 整体风格清新、年轻、校园化；
2. 不要做成死板后台；
3. 主色使用绿色或青绿色，贴合农业大学校园风格；
4. 页面要有卡片感、圆角、轻阴影、渐变背景；
5. 活动大厅要像校园活动平台；
6. 后台管理要清晰高效；
7. 抽签分组页面要有活动工具箱的感觉；
8. 反馈统计页面要有数据分析感；
9. 移动端要适合微信内打开；
10. 保持现有功能可用，不改接口路径。

## 六、组件设计规范

### 1. 按钮

- 主按钮：绿色实心，圆角胶囊，左侧可带图标；
- 次按钮：白底绿字，浅绿色边框；
- 危险按钮：红色文本或浅红底，不要大面积红色；
- H5 底部关键操作可使用全宽主按钮。

### 2. 卡片

- 白底或半透明白底；
- 顶部可加 4-6px 渐变色条；
- 鼠标悬停轻微上移 2-3px；
- 移动端卡片不悬浮，只保留轻阴影。

### 3. 标签

- 校区标签：浅绿底深绿字；
- 状态标签：按状态色区分；
- 类型标签：图片、视频、二维码、抽签、分组使用小图标 + 文本。

### 4. 表单

- 表单不要像后台密集列表；
- 每组表单放在卡片内；
- 表单分组建议：基础信息、时间地点、媒体素材、高级设置；
- 输入框圆角 12px；
- 上传区域使用虚线浅绿边框。

### 5. 表格

- 后台可以保留表格，但外面要包卡片；
- 表头浅绿背景；
- 操作按钮按功能分组，避免一行十几个按钮拥挤；
- 移动端表格改为卡片列表。

## 九、手机端适配方案

断点：

```text
>= 1024px: 桌面双栏
768-1023px: 平板单栏或弱双栏
< 768px: 手机单栏
```

手机端规则：

- 页面左右边距 12-16px；
- Hero 高度降低，标题不超过 30px；
- 活动详情海报不 sticky，变成顶部海报；
- 后台表格改卡片列表；
- 操作按钮尽量全宽或两列；
- 抽奖舞台放在控制面板上方或下方，避免左右布局；
- 上传预览图片 2 列；
- 视频宽度 100%，保持 16:9；
- 顶部导航横向滚动，不强行折行；
- 微信 H5 中底部关键按钮可以固定在底部安全区上方。

## 十一、具体实现清单

### 1. 活动大厅 `/activities`

- 保留现有筛选接口；
- 改为 Hero + 筛选卡片 + 活动卡片网格；
- 活动卡片展示封面、状态、校区、地点、时间、人数和按钮。

### 2. 活动详情 `/activities/:id`

- 使用 `coverUrl` 做左侧海报背景；
- 调用 `GET /api/activities/:id/media`；
- 将媒体分为图片和视频；
- 图片用 `el-image` preview；
- 视频用 `video controls`。

### 3. 活动创建 / 编辑

- `coverUrl` 跟随活动表单保存；
- 活动媒体上传后调用 `/api/activities/:id/media` 保存；
- 新建活动时先创建活动，再保存媒体；
- 编辑活动时加载已有媒体。

### 4. 报名 / 签到

- 保持原接口；
- 增加活动摘要卡和状态反馈；
- 失败提示保持 Element Plus message。

### 5. 后台活动管理

- 保留表格；
- 桌面端表格 + 分组操作按钮；
- 手机端用 CSS 或条件布局改为活动管理卡片。

### 6. 抽签分组

- 保持现有接口；
- 视觉改为工具箱布局：左控制、右舞台、下结果；
- 抽取过程增加轻量动画，不影响业务。

### 7. 反馈统计

- 保持 ECharts；
- 增加反馈附件展示；
- 匿名只隐藏姓名，不隐藏附件。

### 8. 文件 URL

- 后端返回 `/uploads/...` 时，前端不要写死 localhost；
- 开发环境通过 Vite `/uploads` 代理；
