# 数据库设计

默认数据库名：`activity_cube`。

完整建表和种子数据见：`docs/sql/activity_cube.sql`。

## 核心表

- `user`：用户账号、角色、校区、学院班级信息。
- `activity`：活动基础信息、校区、时间、人数上限、跨校区规则、创建人。
- `registration`：报名记录，同一用户同一活动唯一。
- `checkin`：签到记录，同一用户同一活动唯一。
- `feedback`：反馈记录，同一用户同一活动唯一。
- `operation_log`：操作日志，当前 MVP 仅建表预留。

## 多校区

活动校区支持：全校区、龙子湖校区、文化路校区、许昌校区、线上。

用户校区支持：龙子湖校区、文化路校区、许昌校区。
